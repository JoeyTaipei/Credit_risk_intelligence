# 為什麼單一模型不夠——三個盲點

**① 靜態快照** 「逾期 2 次」——三年前的舊帳與最近三個月的連續惡化，模型看到的是同一個數字。

**② 獨立評分** 每個借款人單獨計算——同一擔保圈的群聚詐欺、風險傳染，結構上完全偵測不到。

**③ 文字棄置** 申請書自述、通話紀錄、備註欄——銀行每天收，傳統模型全部丟掉。

> **演講稿（40 秒）**
>
> 這三個盲點不是調參數能解決的，它們是單一模態的結構性限制。
>
> 同樣逾期兩次，走勢惡化和走勢改善是截然不同的風險——表格模型看不到這個區別。一個借款人個人財務數字正常，但他的擔保人背後的圈子正在連環出問題——這個傳染路徑，表格模型也看不到。申請書裡的自述，本身就是信號，現在完全被浪費了。
>
> 我們同時解決這三個問題。

# 系統架構——四個 Encoder，一個決策

![請插入 architecture_overview.png]

**每個 Encoder 把資料壓縮成 32 維向量 → 四個向量拼接成 128 維 → MLP 輸出違約機率 → SHAP + GenAI 報告 → Dashboard**

> **演講稿（50 秒）**
>
> 整個系統的邏輯，從左讀到右。
>
> Encoder 的工作：把各種形式的資料——表格、時序、圖、文字——壓縮成電腦能計算的數字向量。每個 Encoder 輸出一個 32 維的借款人側寫，格式統一，才能拼接在一起。
>
> 四個 32 維向量合成 128 維，進入三層神經網路，輸出違約機率。接著分兩條路：SHAP 解釋加 GenAI 報告，以及直接進 Dashboard。
>
> 最關鍵的設計性質：每個 Encoder 只承諾一件事——輸出 32 維。任何一個 Encoder，我都可以單獨換掉，下游完全不需要改。合成資料換成真實資料，插進來就好。

# 表格 + 時序：XGBoost 與 Bi-LSTM

**XGBoost（表格）**
葉節點 Embedding：每棵樹把借款人分配到一個葉節點，同一葉節點 = 信用行為等價。把葉節點位置轉成 32 維向量，繼承 XGBoost 學好的分群邏輯。

**Bi-LSTM（時序）**
輸入：每位借款人 12 個月 × 4 個特徵。取**最後一個隱藏狀態**而非均值——LSTM 的遺忘門已決定哪些月份值得記住，是有取捨的摘要，不是 12 個月均等平均。

為什麼不用 Transformer？序列只有 12 步，Transformer 的優勢在長序列，這個規模不值得承擔那個複雜度。

> **演講稿（40 秒）**
>
> 表格用 XGBoost，時序用雙向 LSTM。
>
> XGBoost 的技巧：我不只要它的機率輸出，我要它的分群邏輯。每棵樹的葉節點代表「這個借款人跟哪群人最像」，把它轉成向量，就能帶著這個分群知識進入融合層。
>
> LSTM 取最後的隱藏狀態，因為 LSTM 本身在計算中已決定哪些月份重要——均值池化會把這個工作全部丟掉。12 個月的序列，不值得換成 Transformer。

---

# 圖 + 文字：GraphSAGE 與 sentence-BERT

**GraphSAGE（借款人圖）**
Cosine 相似度 ≥ 0.85 連邊，840 節點，10,408 條邊。兩層：第一層看直接鄰居，第二層看「鄰居的鄰居」——這是群聚詐欺的傳染信號。選 GraphSAGE 不選 GCN，因為它學的是聚合函數，新借款人不需要重訓整個圖。*(Stanford CS224W 研究室，Hamilton et al., 2017)*

**sentence-BERT（申請文字）**
凍結 `all-MiniLM-L6-v2` 的 2,200 萬個參數，只訓練一個 384 → 32 維的線性投影頭（12,320 個參數）。Transformer 用 10 億句子對訓練出的語意理解，不能因為 5,000 筆合成資料就覆寫。投影頭學的是：哪個語意方向跟信用違約有關。

> **演講稿（45 秒）**
>
> 圖模組選 GraphSAGE，因為它學的是聚合函數，新借款人直接套函數拿 Embedding，不用重訓。兩層的意義是看到鄰居的鄰居——群聚詐欺的傳染信號，就在這個 2-hop 距離裡。
>
> 文字模組的關鍵決策：Transformer 全部凍住，只訓練一個小投影頭。原因——用 5,000 筆合成資料去更新 2,200 萬個參數，等於用小冊子覆寫圖書館。保留通用語意理解，讓投影頭去學哪個方向跟違約有關。

# Late Fusion——為什麼在決策層才合併

**機制：** 四個 32 維 Embedding → 128 維拼接 → 三層 MLP → 違約機率

**三個理由選 Late，不選 Early：**
- **保留各自強項** 決策樹靠特徵分裂，LSTM 靠閘門，GNN 靠鄰域聚合，Transformer 靠 Attention——輸入層合併會毀掉這些特長
- **模態缺失不崩潰** 某借款人缺少某種資料？補零向量，系統照常輸出
- **獨立維護** 各模型程式碼完全分開，LSTM 出問題不影響 XGBoost

> **演講稿（35 秒）**
>
> Late Fusion 是整個系統最重要的架構選擇：讓每個模型做它最擅長的事，只在最後才把結果合在一起。
>
> 三個好處——保留各自的學習方式、資料缺失時優雅降級、每個模型可以獨立維護。這在需要長期運作的生產系統裡，價值非常大。

# 實驗結果——三個數字的真實意義

| 模組 | Val AUC | 真正代表什麼 |
|---|---|---|
| XGBoost | **0.67** | 唯一碰到真實資料的模組，但 1,200 筆樣本、正例率 40%（真實 6.7%）——分佈失真，完整 15 萬筆預期 0.85+ |
| Bi-LSTM | **0.72** | 合成時序：模型學到的是資料產生器邏輯，不是真實還款行為。自己出題，自己考 |
| GraphSAGE | **0.74** | 合成相似圖：學到的是 Cosine 圖結構，不是信用傳染路徑。同上 |

**AUC 白話：** AUC = 0.67，是「隨機抽一個違約者與一個正常人，模型有 67% 機率把違約者排前面」。不是答對率，是排名能力。

**三個數字不能互相比較。Pipeline 能跑通是唯一被驗證的事。架構才是貢獻。**

> **演講稿（40 秒）**
>
> 這張表，我直接說清楚。
>
> 只有 XGBoost 的 0.67 碰到真實資料——但樣本太小、分佈失真，所以偏低。完整資料集業界 Benchmark 通常在 0.85 以上。LSTM 的 0.72 和 GraphSAGE 的 0.74，是合成資料上的成績，等於用自己出的考題考自己——數字高，不代表更準，代表學到了我寫資料的方式。
>
> 這份簡報的貢獻是架構，不是這幾個數字。

# 可解釋性——三層鏈路，每一層都可審計

![請插入 shap_summary_day1.png]

**第一層 — 特徵 SHAP（XGBoost）**
這張圖顯示整個模型對所有借款人的特徵貢獻分佈。**循環信用使用率**是最大風險驅動因子——高使用率（紅點）一致地把風險分數推高；年齡區段和負債比率緊跟在後。

**第二層 — 模態 SHAP（Fusion）**「時序貢獻 40%；表格 35%；圖 20%；文字 5%」——哪種資料主導了這次判定

**第三層 — GenAI 報告（受控）**
語言模型把兩層 SHAP 數字翻譯成信貸員看得懂的中文報告。Prompt 限制：**只能用 SHAP 已給出的事實，不能自行推測**——防止 AI 胡說的護欄。

> **演講稿（50 秒）**
>
> 可解釋性是這個系統最容易被低估的部分。
>
> 看這張 SHAP 圖。每一列是一個特徵，每一個點是一個借款人，顏色代表特徵值的高低——紅色代表高，藍色代表低。點在零軸右邊，代表這個特徵把風險分數往上推；在左邊，代表往下拉。
>
> 最重要的發現：循環信用使用率的紅點大量落在右側——信用卡用超過、循環信用高的人，風險分數被系統性地推高。這個行為，跟信貸員的直覺完全一致。年齡區段、負債比率緊跟在後，這三個特徵構成了最主要的風險信號。
>
> 第三層 GenAI 報告，是請語言模型把這些數字寫成白話文，但規定它只能用 SHAP 算好的事實說話。這是讓 AI 可以進入合規場景的方式——有根據，可追責。

# 合規設計——可解釋，也可追責

**① Adverse Action（不利行動通知）**
SHAP 直接填入拒貸通知：「主因：循環信用使用率過高（+0.28）；其次：年齡區段（+0.24）」——特徵名、方向、數字，一條都不少。

**② 公平性監控**
年齡、家庭人數可能是敏感族群的代理變數。定期 Fairness Audit：分群檢查核准率與 SHAP Top 特徵，確認沒有系統性偏差。

**③ Audit Trail**

| 欄位 | 說明 |
|---|---|
| `model_version` | 哪個版本做的決定 |
| `shap_top_features` | 前 3 名驅動特徵與數值 |
| `override_reason` | 人工覆蓋時的原因 |
| `reviewer_id` | 執行覆蓋的信貸員 ID |

> **演講稿（40 秒）**
>
> 三件事。
>
> 第一，拒貸理由。這個系統可以直接產出「主因是循環信用使用率過高、貢獻了 +0.28 分」這樣的陳述——符合金融消費者保護法的要求，信貸員可以直接拿去用。
>
> 第二，公平性。年齡和家庭人數可能是代理變數，需要定期做分群審查，確認不同族群的核准率是一致的。
>
> 第三，Audit Trail。每筆決策記錄模型版本、SHAP 前三名、有沒有人工覆蓋、覆蓋理由。`override_reason` 讓人工干預有跡可查，也讓系統和人工判斷可以互相學習。

# 限制與下一步

**目前合成資料的三個已知弱點：**
- **時序** — 逾期事件獨立抽樣，缺乏真實連環效應
- **圖** — 邊代表特徵相似，不是真實擔保關係
- **文字** — 模板生成，語意區辨能力被低估

**真實資料進來，按此順序升級：**
1. **文字** → 真實申請書 + FinBERT，潛力最大
2. **圖** → CRM 真實關係邊，GraphSAGE 才能真正偵測群聚詐欺
3. **時序** → 真實帳務 + 嚴格時間序列分割
4. **表格** → 在本地客群重新校準，美國資料 ≠ 台灣保險客群

**架構已就位。它等待的，是你們的資料。**

> **演講稿（30 秒）**
>
> 直說：三個合成模態的 AUC 不代表真實預測能力。但架構是真實的。每個 Encoder 的 32 維介面、Late Fusion 設計、雙層 SHAP 鏈路——真實資料一進來就能接上，不需要重寫。文字最優先，接著是圖，再來是時序，最後表格重新校準。
